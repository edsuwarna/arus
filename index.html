Features

Everything you need for
data ingestion on a VPS.

Arus gives you the observability of Airflow with the simplicity of a single docker compose up. No Kube, no Kafka, no bloat.

CDC & ETL

Watermark-based incremental extraction from MySQL, MariaDB, and PostgreSQL. Direct (typed) or Raw (JSONB) load modes.

Auto-discover Tables

Register a source DB and Arus automatically discovers all tables, detects sync modes, and suggests configurations.

Visual DAG

Interactive 3-layer asset graph (Source → Raw → Target) with color-coded node statuses and zoom/pan controls.

Scheduling

APScheduler with per-pipeline cron expressions. Set it and forget it — automatic syncs with retry logic.

Schema Drift Detection

New columns in source detected automatically. Raw zone (JSONB) never breaks. Target zone auto-ALTERs.

Alerting

Telegram, Discord, or Slack notifications on pipeline failures, schema changes, and data quality breaches.

Run History & Logs

Full audit trail per pipeline run. View logs, row counts, and durations without SSH access.

Dead Letter Queue

Failed rows are captured and stored. Review, reprocess, or debug without losing data.

How It Works

Three steps to flowing data.

From zero to a running pipeline in under 5 minutes.

Connect

Add your source database — just a name, type, and connection string. Test the connection, then let Arus auto-discover all tables.

Configure

Select which tables to sync, choose incremental or full refresh, pick your target schema, and set the schedule.

Monitor

Watch your data flow through the DAG dashboard. Check run history, inspect logs, get alerts — all from the browser.

Deploy

One command to start.

Arus runs entirely in Docker Compose. Clone the repo, configure, and go.

$ git clone https://github.com/edsuwarna/arus.gitCopy
$ cd arus && docker compose up -dCopy

3 containers · 1 command · zero infrastructure
